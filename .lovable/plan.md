
# Ограничить редактирование отправленного ответа одним разом

## Суть

Wildberries позволяет изменить ответ только 1 раз. Нужно заблокировать возможность повторного редактирования после того, как ответ уже был отредактирован (`is_edited = true`).

## Что будет изменено

### Файл: `src/components/ReviewCard.tsx`

- Если `isEdited === true`, блок отправленного ответа становится **неинтерактивным**:
  - Убрать `cursor-pointer` и `onClick` с текста ответа
  - Не показывать подсказку "Нажмите, чтобы изменить ответ"
  - Не показывать кнопки "Перегенерировать" и "Редактировать"
- Если `isEdited === false` -- поведение остается как сейчас (клик раскрывает кнопки редактирования)

## Логика

```text
isEdited = false  -->  клик на ответ --> кнопки "Перегенерировать / Редактировать"
isEdited = true   -->  текст ответа отображается статично, без возможности клика
```

## Затрагиваемый файл

| Файл | Изменение |
|------|-----------|
| `src/components/ReviewCard.tsx` | Условная блокировка интерактивности при `isEdited === true` |

## Бэкенд

Изменения не требуются -- вся логика на стороне UI.
