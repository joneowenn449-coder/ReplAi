
# Компактное окно настроек

## Проблема

Диалог настроек слишком высокий и обрезается на экране (видно на скриншоте -- не видно кнопок "Отмена/Сохранить" внизу).

## Решение

Сделать диалог компактнее и добавить прокрутку:

### Изменения в `src/components/SettingsDialog.tsx`

1. **Добавить ограничение по высоте и прокрутку** -- обернуть контент в `ScrollArea` или задать `max-h` + `overflow-y-auto`, чтобы при маленьком экране контент прокручивался.

2. **Уменьшить отступы между секциями** -- `space-y-6` заменить на `space-y-4`.

3. **Уменьшить высоту строк рейтинга** -- убрать лишний padding (`py-2` на `py-1.5`), сделать звёзды чуть мельче (`w-4 h-4` на `w-3.5 h-3.5`).

4. **Уменьшить textarea промпта** -- `min-h-[160px]` заменить на `min-h-[100px]`.

5. **Уменьшить padding диалога** -- через класс `DialogContent` задать `p-4` вместо дефолтного `p-6`.

6. **Добавить `max-h-[85vh] overflow-y-auto`** на контейнер контента, чтобы длинное содержимое прокручивалось, а кнопки "Отмена/Сохранить" оставались доступными.

### Технические детали

Файл: `src/components/SettingsDialog.tsx`

Основные CSS-изменения:
- `DialogContent`: добавить `p-4 sm:p-5` для уменьшения внутренних отступов
- Внутренний `div`: добавить `max-h-[70vh] overflow-y-auto pr-1` для прокрутки контента (кнопки вынести за эту область)
- Секция рейтингов: `py-1.5` вместо `py-2`, звёзды `w-3.5 h-3.5`
- Textarea: `min-h-[100px]` вместо `min-h-[160px]`
- Общие отступы: `space-y-4` вместо `space-y-6`
- Кнопки "Отмена/Сохранить" останутся за пределами скроллируемой зоны, всегда видимы внизу
